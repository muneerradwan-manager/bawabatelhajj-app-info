البنية التقنية والمعمارية

Clean Architecture مع BLoC Pattern وفصل واضح للطبقات

4
💡

Clean Architecture — لماذا؟ يتبع التطبيق مبادئ Clean Architecture لفصل المسؤوليات بشكل واضح، مما يجعل اختبار كل طبقة مستقلاً، وإضافة ميزات جديدة أسرع، وصيانة الكود أسهل على المدى الطويل.

طبقات المعمارية لكل ميزة
📱 طبقة العرض (Presentation Layer)
المسؤولية: عرض البيانات للمستخدم والتعامل مع تفاعلاته
المكونات: Screens (الشاشات) + Cubits (إدارة الحالة) + States (حالات الواجهة)
مثال: ComplaintsScreen → ComplaintsCubit → ComplaintsState
⚙️ طبقة المجال (Domain Layer)
المسؤولية: منطق العمل الأساسي المستقل عن أي تقنية
المكونات: Entities (النماذج) + Use Cases (حالات الاستخدام) + Repository Interfaces
مثال: ComplaintEntity + GetComplaintsUseCase + IComplaintsRepository
🗄️ طبقة البيانات (Data Layer)
المسؤولية: جلب البيانات من المصادر (API, Cache, Local)
المكونات: Repository Implementations + Data Sources + DTOs
مثال: ComplaintsRepositoryImpl → ComplaintsApiDataSource + ComplaintsCacheDataSource

بوصلة القبلة الذكية

خوارزمية متطورة تستخدم حساب المسافة Haversine لتحديد اتجاه الكعبة بدقة عالية مع تنعيم القراءة باستخدام خوارزمية Low-Pass Filter (alpha=0.3) لمنع الاهتزاز، ومعالجة ذكية لحدود 0/360 درجة.

📴

العمل بدون إنترنت (Offline-First)

تصميم يُقدّم الاتصال المنقطع كحالة طبيعية: الشكاوى مخزنة محلياً، بيانات الصلاة محفوظة في ملفات JSON، وعداد الصلاة يستمر بدون شبكة. المزامنة تلقائية عند عودة الاتصال.

🏅

الشهادة الرقمية بـ QR

نظام ذكي يلتقط المكوّن كصورة عالية الدقة مع تضمين رمز QR يحتوي على بيانات الحاج والشهادة، مع إمكانية حفظها في المعرض أو مشاركتها مباشرة.

🔄

التحديث الفوري (OTA)

تقنية Shorebird تتيح دفع تحديثات كود Dart مباشرة للمستخدمين دون المرور بمراجعة المتجر، مما يعني إصلاح الأخطاء الحرجة في ساعات لا أيام.
